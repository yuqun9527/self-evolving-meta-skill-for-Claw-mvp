### 为什么这是创新？

传统开发是「人写代码，工具为辅」。这里反转了关系：**Agent 写代码，人只负责定义知识**。

| 传统模式 | 元技能闭环 |
|----------|-----------|
| 每次开发从零开始 | 知识持续积累，越用越强 |
| Bug 修复是人的工作 | 修复自动回写文档，永久免疫 |
| 质量靠人工审查 | 测试套件自动收敛，以数据说话 |
| 技能是静态的 | 技能是**进化的**——每轮迭代都是一个版本 |

**本质**：这不是一个开发工具，而是一个**自我改进系统**。三个技能的真正价值不在「开发出项目」，而在「让开发技能持续进化，直到任何 Agent 按文档都能一次做出正确的项目」。

### 2. claw-mvp-iterate — 迭代引擎（动力源）

> 纯流水线。不知道目标系统叫什么，只知道"创建→测试→修复→循环"。

| 属性 | 值 |
|------|-----|
| 路径 | `claw-mvp-iterate/` |
| 参考文档 | 2 个 |
| 触发场景 | "迭代优化" / "功能测试" / "安全测试" |

```
功能迭代（流水线 A）              安全迭代（流水线 B）
┌──────────────────┐        ┌──────────────────┐
│ 读文档 → 提取功能  │        │ 读文档 → 推导风险  │
│ 设计用例 → 执行测试 │        │ 设计攻击 → 模拟攻击 │
│ 失败 → 修复 → 回归  │        │ 漏洞 → 加固 → 验证  │
│ 全过 → 归档 → 退出  │        │ 全过 → 归档 → 退出  │
└──────────────────┘        └──────────────────┘
```

**核心规则**：
- ⚡ 从目标技能文档提取**所有**领域信息，零硬编码
- 🔢 每轮迭代新目录，序号递增，不覆盖历史
- 📝 发现的 Bug / 教训**强制回写**目标技能文档
- 🧪 测试独立于 LLM API，纯代码驱动
- ⚠️ 同 Bug 3 轮未解 → 报告用户；累计 20 轮 → 暂停

### 3. dev-team-workflow — 协作编排（分工制）

> 三角色双循环。人不用管谁做什么——编排规则自己跑。

| 属性 | 值 |
|------|-----|
| 路径 | `dev-team-workflow/` |
| 参考文档 | 2 个 |
| 触发场景 | "开发项目" / "多角色协作" / "设计→开发→测试" |

**双循环机制**：

| 阶段 | 角色 | 产出 | 循环上限 |
|------|------|------|----------|
| 设计-审查 | Designer ⇄ Developer | `01-设计文档.md` + `01-审查意见.md` | 5 轮 |
| 编码实现 | Developer | 源码 + `02-开发说明.md` | — |
| 开发-测试 | Developer ⇄ Tester | `03-测试报告.md` | 10 轮 |

**角色严格边界**：

| 角色 | 只做 | 绝不 |
|------|------|------|
| 🏗️ Designer | 方案设计、技术选型、按审查修订 | 写代码、做测试 |
| 💻 Developer | 审查设计、编码实现、自测、修 Bug | 改设计方案 |
| ✅ Tester | 编用例、执行测试、输出 Bug 清单 | 改源码、改设计 |

## 🔗 使用指南

### 🚀 场景一：文档齐全，快速构建验证

```
→ 直接启动 claw-mvp-iterate
→ 读取 claw-mvp 全部文档
→ 一次性构建 + 全量测试
→ 自动修复 + 归档
```
**适合**：经过多轮迭代打磨后的成熟技能，文档已收敛。

### 🏗️ 场景二：全新项目，多角色严格把关

```
→ 启动 dev-team-workflow
→ Designer 出设计 → Developer 审查 → 循环到一致
→ Developer 编码 + 自测
→ Tester 测试 → Developer 修复 → 循环到全通过
```
**适合**：首次开发、需求复杂、需要角色制衡的质量关键项目。

### 🔒 场景三：独立安全审计

```
→ 启动 claw-mvp-iterate 流水线 B
→ 从架构推导攻击面 → 加载安全用例
→ 执行攻击模拟 → 加固 → 验证
```
**适合**：定期安全评估、新漏洞变体响应。

## ⚙️ 设计原则

| 原则 | 说明 |
|------|------|
| **文档即代码** | 技能只含模板和规则，不含实现；Agent 从文档生成一切 |
| **零领域耦合** | iterate 和 workflow 不预设目标项目类型，纯通用引擎 |
| **归档即进化** | 迭代发现强制回写文档，知识随轮次增长 |
| **分层可组合** | 知识 / 流程 / 协作三层独立，按需叠加 |
| **以收敛为终** | 不是"功能做完了"，而是"测试不报 Bug 了"才停 |
