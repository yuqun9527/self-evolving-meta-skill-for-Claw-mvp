# 功能迭代 — 通用模板与方法论

本文档只包含通用模板、执行流程和方法论。**不包含任何具体项目的功能清单或测试用例**——这些从目标技能文档中提取。

---

## 一、功能提取方法

### 原则

1. **递归读取目标技能目录下的全部文档**
2. **从中提取系统能力的描述**：工具/模块/入口/格式支持/配置项等
3. **按文档中实际出现的模块分类**，不要套用预设分类
4. **输出功能矩阵**，每个子功能一行

### 功能矩阵模板

| 模块 | 子功能 | 描述 | 优先级 |
|------|--------|------|--------|
| {从文档提取的模块名} | {子功能名} | {一句话} | P0/P1/P2 |

## 二、测试用例模板

### 模板 A：工具/函数测试

```
编号: FUNC-{模块}-{序号}
目标: {一句话}
前置: {准备条件}
输入: {参数/指令}
预期: {结果}
通过: {判定标准}
```

### 模板 B：交互/端到端测试

```
编号: FUNC-E2E-{序号}
目标: {一句话}
步骤:
  1. {操作}
  2. {验证}
预期: {每步预期}
通过: {判定标准}
```

### 模板 C：行为/状态测试

```
编号: FUNC-BEH-{序号}
目标: {一句话}
场景: {上下文}
触发: {动作}
预期行为: {系统应如何反应}
通过: {可观测的验证点}
```

## 三、执行流程

```
1. 提取功能清单（从目标技能文档）
2. 设计测试用例（每子功能 ≥ 1 个）
3. 排序（P0 → P1 → P2）
4. 逐个执行：
   ├─ 清理环境
   ├─ 准备数据
   ├─ 运行
   ├─ 验证结果
   └─ 记录
5. 汇总：通过数 / 失败数
6. 失败处理：
   ├─ 定位代码级根因
   ├─ 修复
   ├─ 回归
   └─ 归档到目标技能文档
7. 决策：全过→退出 / 有修复→下一轮
```

## 四、测试脚本编写指南

### 不依赖 LLM 的独立测试

纯代码逻辑（参数校验、安全检查、格式检测等）可直接写脚本测试：

```
// 通用模式：
// 1. 导入被测试的函数
// 2. 准备测试用例数组 [{input, expect}, ...]
// 3. 遍历执行，对比预期
// 4. 输出通过/失败统计
```

### 依赖 LLM 的端到端测试

需要 LLM 参与的场景（多轮调用、行为验证），启动系统后发送自然语言指令，验证完整链路。

## 五、测试分类参考

| 类别 | 关注点 | 典型验证 |
|------|--------|---------|
| 正常路径 | 合法输入 | 正确输出 |
| 边界条件 | 空值/超限/极端 | 不崩溃，合理降级 |
| 错误处理 | 非法输入 | 明确 error，非静默失败 |
| 安全检查 | 攻击载荷 | 被拦截，不泄露 |
| 编码兼容 | 特殊字符 | 无乱码 |
| 并发隔离 | 多会话 | 数据不串 |
